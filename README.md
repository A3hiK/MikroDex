## 🖥️ Overview

**MikroDex** brings a professional, glassmorphism-powered interface to MikroTik hotspot management. Whether you're an ISP technician or a network admin, MikroDex gives you full control — beautifully.

<br/>

### 💡 Why MikroDex?

| Problem | MikroDex Solution |
|---|---|
| 😤 Winbox is outdated & clunky | ✅ Modern dark UI with smooth animations |
| 😤 No real-time monitoring | ✅ Live dashboard, auto-refresh every 10s |
| 😤 No local data history | ✅ SQLite — full session & usage history |
| 😤 Slow user management | ✅ One-click create, edit & delete users |
| 😤 No offline access | ✅ Works fully offline, no cloud needed |

## 🚀 Getting Started

### ✅ Prerequisites

- **Node.js** v16 or higher → [Download](https://nodejs.org/)
- **npm** or **yarn**
- A **MikroTik router** with REST API enabled on port `8728`

<br/>

### 📦 Installation

**1. Clone the repository**

```bash
git clone https://github.com/A3hiK/MikroDex.git
cd MikroDex
```

**2. Install dependencies**

```bash
npm install
```

**3. Start in development mode**

```bash
npm run electron-dev
```

> 💡 This starts the Vite dev server on `http://localhost:5173` and launches the Electron app with hot reload.

**4. Build for production**

```bash
npm run electron-build
```

> 📦 Windows installer will be generated in the `out/` directory.

## 🔌 MikroTik API Usage

```js
import MikroTikService from './services/MikroTikService';

// Connect to router
const result = await MikroTikService.connect('192.168.88.1', 'admin', 'password');

// Get all online users
const onlineUsers = await MikroTikService.getOnlineUsers();

// Create a new hotspot user
const newUser = await MikroTikService.createHotspotUser({
  name: 'testuser',
  password: 'pass123',
  profile: 'default'
});

// Delete a user
await MikroTikService.deleteHotspotUser('testuser');
```

**Supported Operations:**

| Operation | Description |
|-----------|------------|
| `connect()` | Establish connection to router |
| `disconnect()` | Close router connection |
| `getHotspotUsers()` | List all hotspot users |
| `createHotspotUser()` | Add a new user |
| `deleteHotspotUser()` | Remove a user |
| `getOnlineUsers()` | Get live active sessions |
| `getRouterInfo()` | Fetch router system details |

## 🐛 Troubleshooting

<details>
<summary><b>🔴 Connection Issues</b></summary>
<br/>

- Verify the router IP address is reachable on your network
- Ensure MikroTik REST API is enabled on port `8728`
  - Go to: **Winbox → IP → Services → api-ssl / api**
- Check firewall rules on both the router and your machine
- Confirm your username and password are correct

</details>

<details>
<summary><b>🔴 Database Issues</b></summary>
<br/>

- To fully reset the database, delete this file:
  ```
  C:\Users\{Username}\.mikrodex\database.db
  ```
- Rebuild the native module:
  ```bash
  npm rebuild better-sqlite3
  ```

</details>

<details>
<summary><b>🔴 Electron / Build Issues</b></summary>
<br/>

- Clear the Vite cache — delete the `.vite` folder
- Reinstall all dependencies from scratch:
  ```bash
  rm -rf node_modules
  npm install
  ```

</details>

## 👨‍💻 Extending MikroDex

### ➕ Adding a New Page

```
1. Create component in  src/pages/YourPage.jsx
2. Import it in        src/App.jsx
3. Add a route in      <Routes>
4. Add menu entry in   src/components/Sidebar.jsx
```

### ➕ Adding a New Service

```
1. Create file in      src/services/YourService.js
2. Export as singleton instance
3. Import and use in your components
```
